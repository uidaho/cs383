Use cases:

Edit Project Permissions
----------------------------------------
Actor: Squire user
Goal: Edit the permissions for a project
Precondition: User must have the project that he wants to edit the permissions of open.
Summary: User opens up the settings menu and navigates to permissions, adds (or removes) users individual access rights to the project. 
Related use: Add user to project, Remove user from project.
Steps:
1. Click the settings button in project.
2. System opens settings menu.
3. User clicks permissions link.
4. System opens the permissions menu.
5. User selects user from list of users.
6. User adds read or write permissions to user.
7. User saves changes and exits permissions.

Alternative: User can remove read or write permission instead in step 6. User can discard changes instead in step 7.


Open User Preferences
----------------------------------------
Actor: Squire user
Goal: Open preferences for current user
Precondition: User must be logged into squire.
Summary: User opens up the settings menu, navigates to preferences.
Related use: Change user color, Change password.

Steps:
1. Click the settings button.
2. System opens settings menu.
3. User navigates to preferences. 
4. System opens preferences menu.

Alternative: None

Edit User Color
----------------------------------------
Actor Squire user
Goal: Change user color from within preferences
Precondition: User must have preferences open.
Summary: User selects their color and saves their changes.
Relates use: Open user preferences.

Steps:
1. User clicks select color.
2. System opens color selection window.
3. User selects their color.
4. System updates and previews color.
5. User saves changes.

Alternative: user can discard changes by canceling on color selection window.


Change user password
----------------------------------------
Actor Squire user
Goal: Change user password from within preferences
Precondition: User must have preferences open.
Summary: User types a new password and saves their changes.
Relates use: Open user preferences.

Steps:
1. User clicks “change password”.
2. System opens password changing window.
3. User types their new password.
4. System requests confirmation.
5. User saves changes.

Alternative: user can discard changes by canceling on confirmation window.
